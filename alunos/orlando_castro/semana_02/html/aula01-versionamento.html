Exemplo Real do Dia a Dia

Cenário: Você está desenvolvendo um aplicativo de pedidos para uma lanchonete.

Segunda-feira: Você cria a tela de login. Faz um commit: "Adiciona tela de login"
Terça-feira: Adiciona o menu de produtos. Commit: "Adiciona menu de produtos"
Quarta-feira: Adiciona carrinho de compras. Commit: "Adiciona carrinho"
Quinta-feira: Você muda a cor do botão e o app para de funcionar! 😱

Com Git: Você simplesmente volta para o commit de quarta-feira (que funcionava) e descobre que a mudança de cor causou um conflito com outro código. Você corrige e faz novo commit!

Total de tempo perdido: 5 minutos
Sem Git: Você gastaria HORAS tentando lembrar o que mudou e refazendo código.

Git ≠ GitHub

A confusão que TODO iniciante faz (e agora você não vai fazer!)

🔧

GIT

  • É um SOFTWARE que você instala no PC
  • Funciona OFFLINE (não precisa de internet)
  • Controla versões LOCALMENTE
  • Criado por Linus Torvalds em 2005
  • É a FERRAMENTA que faz o versionamento
  • Gratuito e open source
☁️

GITHUB

  • É um SITE na internet (github.com)
  • Precisa de INTERNET para funcionar
  • Guarda repositórios na NUVEM
  • Criado em 2008, comprado pela Microsoft
  • É o "Google Drive" para código
  • Permite colaboração entre desenvolvedores

Analogia Perfeita

Git = Microsoft Word

Software instalado no seu computador que você usa para criar e editar documentos localmente.

GitHub = OneDrive

Serviço na nuvem onde você guarda e compartilha esses documentos com outras pessoas.

🌍 Outros serviços similares ao GitHub

• GitLab - Alternativa popular, muito usado em empresas
• Bitbucket - Da Atlassian (mesma empresa do Jira)
• Azure DevOps - Da Microsoft

Mas o GitHub é o mais popular e usado pela comunidade open source!

Conceitos Fundamentais

Os 4 pilares que você precisa dominar

1

📁 REPOSITÓRIO (Repo)

É a PASTA do seu projeto. Dentro dela fica todo o código + histórico de versões.

• Repositório Local = pasta no seu computador
• Repositório Remoto = pasta lá no GitHub (nuvem)

2

📸 COMMIT

É uma "fotografia" do código em um momento específico. Cada commit tem:

• Mensagem explicando o que mudou
• Data e hora
• Nome de quem fez
• Quais arquivos foram alterados

3

🌿 BRANCH (Ramificação)

É como criar uma "cópia paralela" do código para testar coisas novas sem estragar o código principal.

• main = branch principal (código funcionando)
• develop = branch de desenvolvimento
• feature/nova-funcionalidade = branch para funcionalidade específica

4

🔀 GITFLOW

É uma METODOLOGIA de organização de branches. Um jeito padronizado de nomear e usar branches para que todo o time trabalhe de forma organizada e sem conflitos.

Estrutura de Branches (GitFlow)

main
← Código em produção (no ar)
↑ merge
develop
← Desenvolvimento ativo
↑ merge
feature/login
← Funcionalidade específica

1

Baixar o instalador

Acesse: https://git-scm.com/downloads
Clique em "Download for Windows" (ou seu sistema operacional)

Executar instalador

⚠️ IMPORTANTE: Use as opções PADRÃO (recomendadas)

Pronto para usar!

🔄 Fluxo de Trabalho: Fork → PR

Este é o fluxo PROFISSIONAL usado em empresas como Google, Microsoft, Meta. É como projetos open-source trabalham (Linux, Python, React, etc)

1

🍴 Fazer FORK do repositório da turma

No GitHub, clique no botão "Fork" (canto superior direito)
Isso cria uma CÓPIA do repositório da turma na sua conta
Agora você tem: github.com/SEU_USUARIO/turma-visualizacao-dados-bi-t2

↓
2

📥 Clonar SEU FORK para o computador

$ git clone https://github.com/SEU_USUARIO/turma-visualizacao-dados-bi-t2.git
$ cd turma-visualizacao-dados-bi-t2
↓
3

📝 Criar SUA pasta e adicionar código

# Navegue até a pasta alunos
$ cd alunos/
$ mkdir joao_silva # Use seu nome (sem espaços!)
$ cd joao_silva/
# Crie seus arquivos aqui
$ code exercicio_01.py # Abre no VS Code
↓
4

💾 Fazer commit das mudanças

$ git add .
$ git commit -m "Adiciona exercício 01 - João Silva"
$ git push origin main
↓
5

🔀 Criar PULL REQUEST

1. Vá no GitHub, no SEU fork
2. Clique em "Pull Request"
3. Clique em "New Pull Request"
4. Escreva um título claro: "[João Silva] Exercício 01 - Sistema de Cadastro"
5. Na descrição, explique o que você fez
6. Clique em "Create Pull Request"

↓
6

👨‍🏫 Professor revisa e aprova (ou pede correção)

O professor vai:
• Revisar seu código
• Fazer comentários se necessário
• Pedir correções (se houver problemas)
• APROVAR e fazer merge quando estiver OK

Assim que for aprovado, seu código entra no repositório principal! 🎉

Governança: Professor controla qualidade

O que NÃO fazer

❌ Clonar o repositório original (clone do professor)
❌ Tentar fazer push direto na main

📋 COMANDOS ESSENCIAIS DO GIT (RESUMO)
$ git clone URL # Copia repositório do GitHub para PC
$ git status # Mostra arquivos modificados
$ git add . # Prepara TODOS os arquivos
$ git commit -m "Mensagem clara" # Salva versão localmente
$ git push origin main # Envia para SEU fork no GitHub
$ git pull origin main # Baixa atualizações do GitHub
$ git log # Mostra histórico de commits
