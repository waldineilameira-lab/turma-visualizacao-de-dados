📖 Módulo 1 — Git & GitHub Básico

O que é Git?

Git é um sistema de controle de versão distribuído. Pense nele como um "histórico completo" do seu código, permitindo voltar a qualquer versão anterior.

O que é GitHub?

GitHub é uma plataforma na nuvem para hospedar repositórios Git. É como um "Google Drive para código", mas muito mais poderoso.

Conceitos Fundamentais

Repository (Repositório)

É a pasta do seu projeto. Contém todos os arquivos e o histórico completo de mudanças.

Commit

É uma "fotografia" do seu código em um momento específico. Cada commit tem:

  • Uma mensagem descritiva
  • O autor
  • Data e hora
  • As mudanças exatas feitas

Push

Envia seus commits locais para o GitHub (nuvem).

Pull

Baixa as mudanças do GitHub para seu ambiente local.

Branch

Uma "linha paralela" de desenvolvimento. Permite trabalhar em features sem afetar o código principal.

✅ Analogia Útil

Pense no Git como um sistema de "save points" em um jogo. Cada commit é um save point que você pode voltar. O GitHub é onde você guarda esses save points na nuvem.

🔑 Módulo 2 — Configuração do Token GitHub

⚠️ Importante

GitHub não aceita mais senha para autenticação via linha de comando. Você precisa criar um Personal Access Token (PAT).

Passo a Passo para Criar Token

  1. Acesse GitHub Settings

    Vá em: https://github.com/settings/tokens

  2. Generate New Token

    Clique em "Generate new token" → "Generate new token (classic)"

  3. Configure o Token
    • Note: "Colab Data Projects"
    • Expiration: 90 days (ou No expiration se preferir)
    • Scopes: Marque repo (dá acesso completo aos repositórios)
  4. Gerar e Copiar

    Clique em "Generate token" e COPIE o token imediatamente!

🚨 ATENÇÃO

O token só aparece UMA VEZ! Salve em um lugar seguro (gerenciador de senhas, arquivo local, etc.).

Se perder, terá que criar um novo.

Formato do Token

ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

Sempre começa com ghp_ seguido de 40 caracteres.

💻 Módulo 3 — Usando Git no Colab

Configuração Inicial

Toda vez que abrir um novo notebook Colab, execute estas configurações:

# Configurar usuário Git
!git config --global user.name "Seu Nome"
!git config --global user.email "seuemail@exemplo.com"

# Verificar configuração
!git config --global --list
💡 Dica

Use o MESMO email da sua conta GitHub para manter consistência.

Clonar um Repositório

Método 1: Repositório Público

# Clonar repositório público
!git clone https://github.com/usuario/repositorio.git

# Entrar na pasta
%cd repositorio

Método 2: Repositório Privado (com token)

# Clonar usando token
!git clone https://SEU_TOKEN@github.com/usuario/repositorio.git

# Exemplo real:
!git clone https://ghp_abc123xyz@github.com/joao/analise-dados.git
⚠️ Segurança

NUNCA compartilhe notebooks com seu token visível! Considere usar variáveis de ambiente ou Colab Secrets.

Usando Colab Secrets (Recomendado)

from google.colab import userdata
import os

# Buscar token dos secrets
token = userdata.get('GITHUB_TOKEN')

# Clonar usando o token
!git clone https://{token}@github.com/usuario/repo.git

Para adicionar um secret:

  1. Clique no ícone de chave 🔑 na barra lateral esquerda
  2. Clique em "+ Add new secret"
  3. Name: GITHUB_TOKEN
  4. Value: Cole seu token

📊 Módulo 4 — Primeiro Projeto de Dados

Cenário: Análise de Vendas

Vamos criar um projeto completo de análise de dados e versioná-lo com Git.

1. Criar Repositório no GitHub

  1. Acesse GitHub e clique em "New repository"
  2. Nome: analise-vendas
  3. Description: "Análise de dados de vendas com Python"
  4. Escolha "Public" ou "Private"
  5. Marque "Add a README file"
  6. Clique em "Create repository"

2. Clonar no Colab

from google.colab import userdata

# Buscar token
token = userdata.get('GITHUB_TOKEN')
username = "seu_usuario"
repo = "analise-vendas"

# Clonar
!git clone https://{token}@github.com/{username}/{repo}.git

# Entrar na pasta
%cd {repo}

3. Criar Estrutura do Projeto

# Criar pastas
!mkdir -p data notebooks scripts

# Criar arquivo de análise
%%writefile notebooks/analise_inicial.py
import pandas as pd
import matplotlib.pyplot as plt

# Carregar dados
df = pd.read_csv('../data/vendas.csv')

# Análise básica
print(df.describe())
print(f"Total de vendas: R$ {df['valor'].sum():.2f}")

4. Criar Dataset de Exemplo

import pandas as pd
import numpy as np

# Gerar dados fictícios
np.random.seed(42)
df = pd.DataFrame({
    'data': pd.date_range('2024-01-01', periods=100),
    'produto': np.random.choice(['A', 'B', 'C'], 100),
    'valor': np.random.uniform(10, 100, 100),
    'quantidade': np.random.randint(1, 10, 100)
})

# Salvar
df.to_csv('data/vendas.csv', index=False)
print("✅ Dataset criado: data/vendas.csv")

🔄 Módulo 5 — Git Workflow (add, commit, push)

O Ciclo Básico do Git

┌─────────────┐
│ Modificar   │ (Editar arquivos)
└──────┬──────┘
       ↓
┌─────────────┐
│ git add     │ (Preparar mudanças)
└──────┬──────┘
       ↓
┌─────────────┐
│ git commit  │ (Salvar snapshot)
└──────┬──────┘
       ↓
┌─────────────┐
│ git push    │ (Enviar ao GitHub)
└─────────────┘

Comandos Detalhados

1. Verificar Status

# Ver arquivos modificados
!git status

2. Adicionar Arquivos

# Adicionar arquivo específico
!git add notebooks/analise_inicial.py

# Adicionar todos os arquivos
!git add .

# Adicionar por pasta
!git add notebooks/

3. Fazer Commit

# Commit com mensagem
!git commit -m "Adiciona análise inicial de vendas"

# Commit mais descritivo
!git commit -m "feat: análise de vendas por produto

- Carrega dataset de vendas
- Calcula estatísticas descritivas
- Gera gráfico de vendas por produto"

4. Enviar ao GitHub (Push)

# Push para branch main
!git push origin main

Exemplo Completo

# 1. Ver o que mudou
!git status

# 2. Adicionar arquivos
!git add .

# 3. Commit
!git commit -m "feat: adiciona análise inicial de vendas"

# 4. Push
!git push origin main

🌍 Módulo 6 — Local vs Remoto

Entendendo a Diferença

Local (Colab) Remoto (GitHub)
Arquivos no notebook Colab Arquivos no GitHub
Commits salvos localmente Commits visíveis online
Temporário (Colab reseta) Permanente
Só você vê Qualquer um pode ver (se público)

Comandos de Sincronização

Ver repositórios remotos

!git remote -v

Ver histórico de commits

# Histórico completo
!git log

# Histórico resumido
!git log --oneline

🐛 Módulo 7 — Erros Comuns e Soluções

Erro 1: Authentication Failed

Erro: 
remote: Support for password authentication was removed

Solução:

# Use token no lugar da senha
!git clone https://SEU_TOKEN@github.com/user/repo.git

Erro 2: Large Files

Erro: 
this exceeds GitHub's file size limit of 100 MB

Solução:

# Adicionar ao .gitignore
echo "*.csv" >> .gitignore
!git rm --cached arquivo_grande.csv

🌿 Módulo 9 — Branches

O que são Branches?

Branches são "linhas paralelas" de desenvolvimento. Permitem trabalhar em features sem afetar o código principal.

Comandos de Branch

Criar branch

# Criar e mudar para ela
!git checkout -b feature/analise-temporal

Mudar de branch

# Mudar para branch existente
!git checkout feature/analise-temporal

# Voltar para main
!git checkout main

Enviar branch para GitHub

# Push da branch
!git push origin feature/analise-temporal

🔀 Módulo 10 — Pull Requests

O que é Pull Request (PR)?

É uma solicitação para mesclar (merge) mudanças de uma branch para outra. Permite revisão de código antes de incorporar ao projeto principal.

Como Criar Pull Request

  1. Criar e trabalhar em branch 
    !git checkout -b feature/nova-visualizacao
# Fazer mudanças...
!git push origin feature/nova-visualizacao
  2. Ir ao GitHub

    Acesse: https://github.com/usuario/repo

  3. Clicar em "Compare & pull request"
  4. Preencher informações e criar PR

⬇️ Módulo 11 — Git Pull

O que é Git Pull?

git pull baixa e integra mudanças do repositório remoto (GitHub) para seu ambiente local (Colab).

Comandos Pull

# Pull da branch atual
!git pull

# Pull de branch específica
!git pull origin main

👥 Módulo 12 — Colaboração em Projetos de Dados

Workflow em Equipe

  3. Code Review
  4. Merge em develop
  5. Testar integração
  6. Deploy: develop → main
