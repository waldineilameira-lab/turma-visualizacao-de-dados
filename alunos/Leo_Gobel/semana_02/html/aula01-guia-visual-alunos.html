Passo 1: Fazer Fork

Criar sua cópia do repositório

https://github.com/cfneves/turma-visualizacao-de-dados
🍴

Clique no botão FORK

Localizado no canto superior direito da página do GitHub

Fork = Cópia completa do repositório na SUA conta

O que acontece:

1

Antes do Fork

Existe apenas 1 repositório:
github.com/cfneves/turma-visualizacao-de-dados (professor)

↓ você clica em FORK
2

Depois do Fork

Existem 2 repositórios:
github.com/cfneves/turma-visualizacao-de-dados (professor)
github.com/SEU_USUARIO/turma-visualizacao-de-dados (VOCÊ)

✅ Pronto!

Agora você tem sua própria cópia do repositório. Pode mexer à vontade sem afetar o repositório do professor!

Passo 2: Clonar Fork

Baixar o repositório para seu computador

⚠️ ATENÇÃO CRÍTICA

Clone o SEU fork, NÃO o repositório do professor!

❌

ERRADO

$ git clone https://github.com/cfneves/turma-visualizacao-de-dados.git
✗ Este é o repositório do PROFESSOR!
✅

CERTO

$ git clone https://github.com/SEU_USUARIO/turma-visualizacao-de-dados.git
✓ Este é o SEU fork!
# Passo a passo completo:
$ git clone https://github.com/SEU_USUARIO/turma-visualizacao-de-dados.git
# Aguarde o download...
$ cd turma-visualizacao-de-dados
# Agora você está dentro da pasta do repositório!
$ ls
alunos/ aulas/ datasets/ exercicios/ README.md
✅ Resultado

Agora você tem uma pasta no seu computador com todos os arquivos do repositório!

Passo 3: Criar Sua Pasta

Dentro de alunos/ crie sua pasta pessoal

📝 Regras de Nomenclatura

Nome da pasta deve ser: snake_case (minúsculo com underscore)

❌

ERRADO

João Silva → tem espaço
JoaoSilva → PascalCase
joaoSilva → camelCase
joao-silva → kebab-case
JOAO_SILVA → maiúsculo

✅

CERTO

joao_silva ✓
maria_santos ✓
pedro_costa ✓
ana_oliveira ✓

Padrão: primeiro_nome_sobrenome

# Entre na pasta alunos:
$ cd alunos/
# Crie SUA pasta (use seu nome!):
$ mkdir joao_silva
# Entre na sua pasta:
$ cd joao_silva/
# Crie um README.md pessoal:
$ echo "# João Silva - Visualização de Dados" > README.md
✅ Estrutura Criada

Agora você tem alunos/joao_silva/README.md pronto para fazer commit!

Passo 4: Fazer Commit

Salvar suas mudanças localmente

# Volte para raiz do projeto:
$ cd ../..
# Veja o que mudou:
$ git status
Untracked files:
alunos/joao_silva/README.md
# Adicione suas mudanças:
$ git add .
# Faça commit com mensagem CLARA:
$ git commit -m "[João Silva] Cria pasta pessoal e README"
[main a1b2c3d] [João Silva] Cria pasta pessoal e README
1 file changed, 1 insertion(+)
📝 Padrão de Mensagem

[Seu Nome] Descrição clara do que fez

Exemplos:
✅ [João Silva] Adiciona exercício 01
✅ [Maria Santos] Corrige bug no exercício 02
❌ update (muito vago)
❌ exercicio (sem detalhes)

✅ Commit Feito!

Suas mudanças estão salvas LOCALMENTE. Próximo passo: enviar para seu fork no GitHub!

Passo 5: Push (Enviar)

Enviar commit para SEU fork no GitHub

# Envie para SEU fork:
$ git push origin main
# Vai pedir usuário e senha do GitHub...
Enumerating objects: 5, done.
Counting objects: 100% (5/5), done.
Writing objects: 100% (3/3), 345 bytes | 345.00 KiB/s, done.
To https://github.com/joao_silva/turma-visualizacao-de-dados.git
a1b2c3d..e4f5g6h main -> main

O que aconteceu:

1

Antes do Push

Commit está APENAS no seu computador
SEU fork no GitHub ainda está desatualizado

↓ git push
2

Depois do Push

Commit está no seu computador E no seu fork do GitHub
Agora você pode abrir Pull Request!

✅ Código na Nuvem!

Seu código está salvo no GitHub. Mesmo se seu computador quebrar, o código está seguro!

Passo 6: Pull Request

Pedir para o professor aprovar seu código

https://github.com/SEU_USUARIO/turma-visualizacao-de-dados

No SEU fork, clique em:

Pull Request
1

Acesse SEU fork

https://github.com/SEU_USUARIO/turma-visualizacao-de-dados

2

Clique em "Pull Request"

Aparece mensagem: "This branch is 1 commit ahead"

3

Clique em "New Pull Request"

Botão verde no canto superior direito

4

Preencha os campos

Título: [João Silva] Cria pasta pessoal
Descrição: Explique o que você fez

5

Create Pull Request

Clique no botão verde "Create Pull Request"
Pronto! Aguarde aprovação do professor

🎉 Pull Request Criado!

Professor vai revisar seu código e aprovar. Quando aprovado, seu código entra no repositório principal!
