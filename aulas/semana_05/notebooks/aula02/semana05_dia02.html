Bloco 1 · 19h15 groupby — agrupando e resumindo dados

"groupby é o equivalente do GROUP BY do SQL. Você escolhe uma ou mais colunas para agrupar, escolhe uma coluna para calcular, e escolhe a função. O resultado sempre é uma Series ou DataFrame menor que o original — e é exatamente esse resultado que você vai plotar."

EXEMPLO 01groupby com uma coluna e agg() com múltiplas funções

O padrão mais comum: agrupa por uma coluna categórica e calcula várias estatísticas de uma vez com agg().

aula_02.ipynb · Célula 2Python
# Resumo salarial por departamento
resumo = (
    df.groupby("Departamento")["Salario"]
    .agg(
        quantidade = "count",
        media      = "mean",
        mediana    = "median",
        minimo     = "min",
        maximo     = "max"
    )
    .round(2)
    .sort_values("media", ascending=False)
    .reset_index()   # transforma o índice em coluna normal
)

print(resumo.to_string(index=False))

📊 O que o gráfico mostra

Por que reset_index() após groupby

O groupby transforma a coluna agrupada em índice do resultado. Para plotar com plt.barh(dados["Departamento"], ...) você precisa que "Departamento" seja uma coluna normal — não o índice. O reset_index() faz exatamente isso.

📊 O que o gráfico mostra

Produção e RH têm o maior volume de inativos em números absolutos — o que faz sentido por serem os maiores departamentos. Para comparar taxa de inatividade de forma justa entre departamentos de tamanhos diferentes, o próximo passo seria calcular o percentual, não o absoluto.

Por que unstack() para barras agrupadas

unstack() "gira" o segundo nível do índice hierárquico e transforma os valores em colunas separadas. Isso é o que o plot(kind="bar") precisa para criar barras lado a lado: uma coluna por grupo. Sem o unstack, você teria uma série única e uma barra só por linha.

plt.plot() vs plt.bar()

Use plt.bar() para comparar categorias sem relação de sequência. Use plt.plot() quando o eixo x é temporal ou ordinal — o traço entre os pontos comunica continuidade e tendência, que é exatamente o que você quer mostrar numa série histórica.

Bloco 2 · 20h40 merge e pivot_table

"merge é o JOIN do SQL — você cruza duas tabelas por uma coluna em comum. pivot_table é a Tabela Dinâmica do Excel — você cruza duas colunas categóricas e coloca um número no meio. Os dois são muito usados em relatórios de gestão, e os dois têm um gráfico natural que conta a história do dado."

# INNER merge: só departamentos que existem nos dois DataFrames
comparacao = pd.merge(
    resumo[["Departamento", "media"]],
    metas,
    on="Departamento",
    how="inner"
)

# Calcular diferença entre média real e meta
comparacao["Diferenca"] = (comparacao["media"] - comparacao["Meta_Salario"]).round(2)
comparacao["Status_Meta"] = comparacao["Diferenca"].apply(
    lambda x: "✅ Acima" if x >= 0 else "❌ Abaixo"
)

print(comparacao.to_string(index=False))

inner vs left merge

how="inner" retorna apenas departamentos que existem nos dois DataFrames. "Manutenção" sumiu do resultado porque não estava na tabela de metas. Se você quisesse manter todos os departamentos do RH e preencher a meta como NaN onde não existir, usaria how="left".

EXEMPLO 05pivot_table — relatório cruzado em uma linha

pivot_table é o equivalente da Tabela Dinâmica do Excel: você coloca uma coluna no eixo das linhas, outra no eixo das colunas, e agrega um valor no meio.

aula_02.ipynb · Célula 11Python
# Salário médio cruzando Departamento × Cargo
pivot = pd.pivot_table(
    df,
    values  = "Salario",      # o que calcular
    index   = "Departamento",  # linhas
    columns = "Cargo",         # colunas
    aggfunc = "mean",          # função de agregação
    fill_value = 0             # preenche onde não há dado
).round(0)

print(pivot.to_string())

📊 O que o gráfico mostra

A estrutura salarial por cargo é consistente entre departamentos — o que indica uma política de remuneração padronizada. O cargo "Gerente" tem a maior dispersão entre departamentos, sugerindo maior negociação individual nesse nível.

Por que pivot_table é mais rápido que groupby aqui

Para cruzar duas colunas categóricas com uma agregação, pivot_table é mais direto que groupby + unstack. O resultado já vem no formato de tabela bidimensional que o plot(kind="bar") entende nativamente — sem precisar de reshape.
